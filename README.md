# Трекер сна

Локальный дневник и анализ сна — PWA-приложение для Android и iOS, устанавливаемое прямо
из браузера (без APK/IPA и без App Store/Google Play). Все данные хранятся только на
устройстве пользователя (IndexedDB), сервера или аккаунта нет.

## Возможности

- Онбординг из 4 экранов при первом запуске.
- Ежедневная анкета: сколько часов вы спали, насколько крепко, что снилось, самочувствие.
- Мгновенный локальный «балл сна» (0–100) с анимированным кольцом — взвешенная формула
  по часам/качеству/самочувствию/типу сна, без ML и без сети. Текстовый разбор и совет
  дня подбираются по приоритетному фактору (недосып/тревожный сон/поверхностный сон/
  тяжёлое утро/пересып/отличная ночь/ровная ночь).
- Тренды: средний балл и часы сна, график по последним 7 ночам, журнал последних записей.
- Экспорт/импорт данных в JSON — резервная копия на случай переустановки/очистки браузера.
- Ежедневные напоминания «Пора спать» / «Утренний опрос» (best-effort, см. ограничения ниже).
- Установка на главный экран — баннер и подсказка с шагами под конкретный браузер
  (Android/Chrome, iOS Safari, Safari на Mac, остальные).

## Разработка

Требуется Node.js 20+.

```bash
npm install
npm run dev       # http://localhost:5173/fmikn-sleep-tracker/
npm run build     # production-сборка в dist/
npm run preview   # локальный просмотр production-сборки
```

Иконки (`public/icons/`, `public/apple-touch-icon.png`) генерируются автоматически из
`scripts/icon-source*.svg` перед `dev`/`build` (`npm run icons` — запустить вручную).

## Деплой на GitHub Pages

1. В `vite.config.ts` переменная `base` задана как `/fmikn-sleep-tracker/` — если репозиторий
   переименуете, поправьте её на `/<имя-репозитория>/` (для персональной страницы
   `username.github.io` без подпапки поставьте `/`).
2. В настройках репозитория: **Settings → Pages → Source → GitHub Actions**.
3. Запушьте в ветку `main` — workflow `.github/workflows/deploy.yml` соберёт проект и
   опубликует `dist/` на GitHub Pages автоматически.
4. Откройте сайт на телефоне и добавьте на главный экран через меню браузера
   («Добавить на экран» / «Установить приложение») — установки из стора не требуется.

## Ограничение: напоминания на iOS

У приложения нет бэкенда, поэтому настоящий Web Push (с сервера) не реализован
принципиально. Напоминания работают так:

- Пока приложение открыто (в любой вкладке) — надёжно проверяется каждую минуту.
- На Android в установленном PWA дополнительно используется Periodic Background Sync —
  работает в большинстве случаев, даже если приложение закрыто.
- На iOS Safari уведомления в принципе доступны только после установки на экран «Домой»
  (iOS 16.4+), а фоновая доставка ненадёжна — это ограничение платформы, а не приложения.

## Структура

- `src/analysis/score.ts` — формула балла сна, теги, факторный выбор текста разбора/совета.
- `src/db/db.ts` — вся работа с IndexedDB (записи, экспорт/импорт, настройки напоминаний,
  посев демо-записей при первом запуске).
- `src/components/Onboarding/` — приветственная карусель из 4 слайдов.
- `src/components/TodayTab/`, `TrendsTab/`, `MoreTab/` — три вкладки приложения
  («Сон», «Тренды», «Ещё»).
- `src/components/InstallBanner.tsx`, `InstallSheet.tsx` — призыв установить приложение
  и перетаскиваемая шторка с инструкцией под конкретный браузер.
- `src/sw.ts` — кастомный Service Worker (офлайн-кэш + best-effort напоминания).
